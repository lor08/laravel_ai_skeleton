# Gotchas

> Известные подводные камни, которые невозможно понять из кода.
> Ловушки, странности легаси, неочевидные последствия, особенности библиотек.

## Формат

```markdown
### Краткое описание

**Где:** `app/...` или общее
**Что не так:** что увидишь как странность
**Почему так:** причина (история / ограничение)
**Что делать:** обход / правильный подход
**ADR:** если решение записано — ссылка
```

## Примеры (заполни своими)

### Order.total в копейках, не в рублях

**Где:** `app/Models/Order.php`
**Что не так:** `$order->total` — int, например 12345 при цене 123.45 руб
**Почему так:** избегаем float ошибок при подсчётах
**Что делать:** для отображения — `formatMoney($order->total)`; для сохранения — `* 100`

### Поле `created_at` отсутствует в `legacy_users`

**Где:** `legacy_users` таблица
**Что не так:** Eloquent падает при `User::latest()`
**Почему так:** таблица унаследована из легаси-системы
**Что делать:** в `User::class` — `public $timestamps = false` для этой связи / отдельная модель `LegacyUser`

### Soft delete на `customers`, но не на `orders`

**Где:** `customers` vs `orders`
**Что не так:** удалённый клиент → orphaned orders (customer_id указывает на soft-deleted)
**Почему так:** заказы не удаляем по compliance, клиента — можем
**Что делать:** при выборке — `Order::with('customer')->withTrashed('customer')` или явно проверять
