# /quick — Light workflow

Для мелких задач: правка одного-двух файлов, понятный фикс, тривиальная фича.

Аргумент: `$ARGUMENTS` — короткое описание задачи.

## Шаги

1. **Прочитай:**
   - `AGENTS.md` (если ещё не загружен)
   - Релевантные правила (`.ai/rules/...`) по теме задачи
   - Затронутые файлы

2. **План в 5 пунктов:**
   - Что меняем
   - Где (точные файлы)
   - В каком порядке
   - Какие проверки запустим
   - Какую документацию обновим (если что-то)

3. **Покажи пользователю → дождись `ок`.**

4. **Реализуй** — в порядке из плана. После каждого PHP-файла: `<RUN-CMD> ecs check --fix path/to/File.php`.

5. **Проверки:**
   ```bash
   composer style
   composer analyse
   composer arch
   composer test
   ```

6. **Документация** — прогон по чек-листу из `.ai/rules/docs.md`. Если ничего — отметь явно.

7. **Краткий отчёт** (что изменилось + статусы проверок).

## Когда переключиться на `/task`

Если в процессе обнаружилось:
- Затронуто > 5 файлов
- Нужно архитектурное решение
- Ломаются существующие тесты
- Появились варианты, между которыми надо выбирать

→ Останови `/quick`, скажи пользователю «задача больше, чем казалось — переключаюсь на `/task`».
