# /explain — Explain code

Объясняешь, что делает код, без правок.

Аргумент: `$ARGUMENTS` — путь к файлу, или namespace, или имя класса/метода.

## Шаги

1. **Прочитай:**
   - Целевой файл / класс
   - Все классы, на которые он ссылается (одного уровня глубины)
   - Тесты, которые его покрывают
   - `.ai/project/glossary.md` (для терминов)
   - `.ai/project/domain/{relevant}.md` (если есть)

2. **Объясни в формате:**

```markdown
## {Class / File / Method}

### Назначение (1-2 предложения)

### Откуда вызывается
- `path/to/Caller.php:123` — ...
- ...

### Что делает (по шагам)
1. ...
2. ...
3. ...

### Зависимости
- `OtherClass` — зачем
- `Facade::X` — зачем

### Контракт
- **Принимает:** ...
- **Возвращает:** ...
- **Может бросить:** ...

### Покрытие тестами
- `tests/Unit/...Test.php` — что покрывается
- `tests/Feature/...Test.php` — что покрывается
- ❌ не покрыто: ... (если есть)

### Связанные модули / правила / ADR
- `.ai/project/domain/{X}.md`
- `.ai/adr/NNNN-...md` (если применимо)

### Запахи / возможные улучшения (опц.)
- ...
```

3. **Не правь код.** Это команда для понимания. Если по ходу видишь явные проблемы — упомяни в секции «Запахи», но не правь.

4. **Если не понятно** — задай 1–2 уточняющих вопроса пользователю и доспроси.
